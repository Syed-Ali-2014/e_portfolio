Vulnerability Analysis – Literature Review Activity

Completion Requirements:

Using the website(s) assigned to you in Unit 1, carry out the following exercise and answer the questions listed below.

Instructions:

Based on the information identified about your assigned websites, carry out a literature search/ audit on software sites and the national vulnerabilities database to create a baseline audit on potential vulnerabilities with websites.

Learning Outcomes:

  • Identify and analyse security threats and vulnerabilities in network systems and determine appropriate methodologies, tools and techniques to manage and/or solve them.
  • Design and critically appraise computer programs and systems to produce solutions that help manage and audit risk and security issues.
  • Gather and synthesise information from multiple sources (including internet security alerts and warning sites) to aid in the systematic analysis of security breaches and issues.

Reflection:

Reflect on this activity by answering the following questions:

  • Did you have any issues or challenges with the literature search/audit on software sites and the national vulnerabilities database?
  • How did you overcome them?
  • How will they affect your final report?

Record all reflections in your e-portfolio. Find guidance on creating a GitHub account and e-Portfolio in the Module Resources tab.

Assigned Website

http://zero.webappsecurity.com/

Analysis and Responses

Question 1: Did you have any issues or challenges with the literature search/audit on software sites and the national vulnerabilities database?

Answer: One of the primary challenges encountered during the literature search was the vast amount of data available, making it difficult to filter out the most relevant information. Additionally, navigating through different databases and ensuring the accuracy of the information required a meticulous approach. Another challenge was identifying and verifying the credibility of various sources, which required cross-referencing multiple sources to ensure consistency.

Question 2: How did you overcome them?

Answer: To overcome these challenges, I developed a systematic approach to the literature search. This included setting clear criteria for selecting sources, such as prioritizing peer-reviewed journals and reputable security websites. I also used advanced search techniques, such as Boolean operators, to narrow down the search results. Cross-referencing information from multiple sources helped in verifying the accuracy and credibility of the data. Additionally, keeping detailed notes and organizing the information into categories made it easier to analyze and synthesize the findings.

Question 3: How will they affect your final report?

Answer: The challenges and the methods employed to overcome them will enhance the quality of the final report. The systematic approach to the literature search ensures that the information included in the report is relevant, accurate, and credible. The detailed notes and organized data will facilitate a comprehensive analysis, leading to well-informed conclusions and recommendations. This thorough approach will also add depth to the vulnerability analysis, providing a solid foundation for the baseline audit and subsequent evaluations.

Baseline Audit on Potential Vulnerabilities

Based on the information identified from the literature search and the national vulnerabilities database, the following vulnerabilities were confirmed for the website http://zero.webappsecurity.com/:

  • Lack of SSL (HTTPS): The website uses HTTP instead of HTTPS, making it vulnerable to man-in-the-middle attacks.
  • Outdated Software Components: Several software components used by the website are outdated, which could be exploited by attackers. For example, the version of jQuery used is outdated and has known vulnerabilities.
  • SQL Injection: The website is susceptible to SQL injection attacks due to improper validation of user inputs. This was confirmed using tools like SQLMap.
  • Cross-Site Scripting (XSS): The site has XSS vulnerabilities where attackers can inject malicious scripts, confirmed through manual testing and tools like OWASP ZAP.
  • Weak Authentication Mechanisms: The authentication mechanisms in place are not robust enough to prevent unauthorized access. This was verified through password strength analysis and lack of multi-factor authentication.

This baseline audit serves as a foundation for further in-depth vulnerability assessments and penetration testing, ensuring a comprehensive evaluation of the website's security posture.
